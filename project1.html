RF Switch Network

  • ⮜

Objective

The goal for this project was to create a web application capable of remotely controlling the transmission path routing of an RF switch network. The requirement for the web application was to create a control panel with an intuitive user interface. The requirements for the network circuit were to accommodate 12 relay switches with 6 inputs each that would be controlled by Raspberry Pi GPIO pins.

Process

The web application was created using HTML, CSS, and JavaScript for the frontend, and Python Flask for the backend. Name editing features for channels/nodes and a stopwatch to indicate runtime were also added. The selection, naming, and stopwatch functionalities were all implemented using JavaScript while the GPIO pins were all handled with Python.

To accommodate the 12 relay switches, IO expanders were needed because the Raspberry Pi only provided 26 GPIO pins. SPI IO expanders were chosen over I2C because the switch network is a lower power application. The chosen IC (MCP23S08) offers 8 GPIO per chip which means 9 IO expanders would be needed to cover all necessary pins. Because pricing isn’t a limitation for this project, it was decided that using 12 IO expanders would be more ideal for modularizing the circuit design.

When implementing the SPI IO expanders, there were issues in controlling several GPIO expanders simultaneously. To troubleshoot these issues, the analog and digital SPI signals were viewed in an oscilloscope. In the analog signal, it was found that the voltage source entering the IC wasn’t stable, which was solved by adding a decoupling capacitor to each IC. In the digital signal, it was found that data was being sampled on the falling edge as opposed to the rising edge which was fixed by changing the SPI mode in the code.

Result
